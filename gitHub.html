GitHub

GitHub este o platformă pentru dezvoltatori care le permite dezvoltatorilor să creeze, să stocheze, să gestioneze și să partajeze codul lor. Utilizează software- ul Git , oferind controlul versiunilor distribuite de Git plus controlul accesului , urmărirea erorilor , solicitările de funcții software , managementul sarcinilor , integrarea continuă și wiki-uri pentru fiecare proiect.

SETUP

Configurarea informațiilor despre utilizator utilizate în toate depozitele locale:

git config --global user.name „[prenume nume]”
setați un nume care poate fi identificat pentru credit atunci când examinați istoricul versiunilor

git config --global user.email „[valid-email]”
setați o adresă de e-mail care va fi asociată fiecărui marcator istoric

git config --global color.ui auto
setați colorarea automată a liniei de comandă pentru Git pentru o revizuire ușoară

SETUP & INIT

Configurarea informațiilor despre utilizator, inițializarea și clonarea depozitelor

git init
inițializați un director existent ca depozit Git

clona git [url]
preluați un întreg depozit dintr-o locație găzduită prin intermediul URL-ului

STAGE & SNAPSHOT

Lucrul cu snapshot și zona de pregătire Git

git status
afișați fișierele modificate în directorul de lucru, pregătite pentru următoarea dvs. comitere

git add [file]
adăugați un fișier așa cum arată acum la următoarea comitere (etapă)

git reset [file]
dezactivați un fișier, păstrând modificările în directorul de lucru

git diff
diferență de ceea ce este schimbat, dar nu este pus în scenă

git diff --staged
diferă de ceea ce este pus în scenă, dar încă nu s-a comis

git commit -m “[descriptive message]”
comite conținutul tău în scenă ca un nou instantaneu de comitere

BRANCH & MERGE

Izolarea muncii în ramuri, schimbarea contextului și integrarea schimbărilor

git branch
enumerați-vă filialele. un * va apărea lângă ramura activă în prezent

git branch [branch-name]
creați o nouă ramură la commit-ul curent

git checkout
treceți la o altă ramură și verificați-o în directorul dvs. de lucru

git merge [branch]
îmbina istoricul ramurei specificate în cea curenta

git log
afișează toate modificarile din istoria filialei curente

INSPECT & COMPARE

Examinarea jurnalelor, diferențelor și informațiilor despre obiect.

git log
afișează istoricul modificarilor pentru ramura activă în prezent

git log branchB..branchA
arata commit-urile de pe branchA care nu sunt pe branchB

git log --follow [file]
arată commit-urile care au schimbat fișierul, chiar și în rândul redenumirilor

git diff branchB...branchA
arata diferenta dintre ceea ce este in branchA si care nu este in branchB

git show [SHA]
afișeaza orice obiect în Git în format care poate fi citit de om

TRACKING PATH CHANGES

Fișierul de versiuni este eliminat și calea se modifică

git rm [file]
ștergeți fișierul din proiect și etapa de eliminare pentru commit

git mv [existing-path] [new-path]

git mv [existing-path] [new-path]
modificați o cale de fișier existentă și puneți-o in stage mutarea

git log --stat -M
arată toate jurnalele de comitere cu indicarea oricăror căi care s-au mutat

SHARE & UPDATE

Preluarea actualizărilor dintr-un alt depozit și actualizarea depozitelor locale

git remote add [alias] [url]
adăugați o adresă URL git ca alias

git fetch [alias]
luați toate ramurile de la acel remote Git

git merge [alias]/[branch]
îmbina o ramură la distanță în ramura curentă pentru a o actualiza

git push [alias] [branch]
Transmite commit-urile de ramuri locale către ramura depozitului de la distanță

git pull
preluați și îmbinați orice comitere din ramura de urmărire la distanță
